Report Types - User Guide

This page describes the purpose and structure of each report type available in the application.

1. One Wi-Fi AP per Page (one)

Displays one access point (AP) per page. Best suited for individual network analysis.

  • SSID: Name of the Wi-Fi network.
  • MAC Address: Unique identifier of the access point.
  • Manufacturer: Vendor derived from the MAC address.
  • SSID Channels: Channel numbers where the AP broadcasts.
  • Encryption: Security type (e.g., WPA2, Open).
  • Tests: Results of performed security tests, including timestamps.
  • Cracked Passwords: Any cracked passwords with their discovery time.
  • Frequency Map: Visual chart of broadcast frequency usage.
  • Location Map: Pinpointed AP location with address lookup.

2. All Wi-Fi APs in One Document (separate)

Presents all access points, each in a separate section with their own map and data.

This view allows comparative inspection of all APs scanned during an audit.

3. Wi-Fi APs on One Map (combined)

Plots all discovered access points onto a single interactive map, with detailed info listed below.

Ideal for geographic visualization of wireless coverage and density.

4. CSV Export (csv)

Generates a downloadable CSV report for offline storage or data processing.

Two sections are included:

  • Devices Table:
    • ID: Internal record ID
    • ssid: Network name
    • mac_address: Hardware address
    • manufacturer: Vendor
    • ssid_channels: Channels used
    • frequency_map: Raw signal data
    • encryption: Security protocol
    • lat_avg, lon_avg: Average location coordinates
  • Tests Table:
    • ID: Test entry ID
    • device_id: Reference to the related device
    • test_type: Type of test performed
    • test_result: Result of the test
    • tested_with_device_id: Related AP, if relevant
    • time_stamp: Time of execution
